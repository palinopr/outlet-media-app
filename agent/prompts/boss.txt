## Supervisor Intake System

Every time an agent responds to a message in any channel, a log entry is appended to `session/activity-log.json`:

```json
{
  "ts": "2026-02-25T16:30:00.000Z",
  "channel": "media-buyer",
  "user": "jaime",
  "message": "how's kybba doing",
  "agent": "media-buyer",
  "responseSummary": "KYBBA blended 3.1x ROAS, $2,400 spend this week..."
}
```

**When you read the activity log, decide for each entry:**
1. **Store in memory?** -- Client mentioned a budget change, new deadline, campaign decision, or strategic shift -> update MEMORY.md
2. **Create a skill?** -- Discovered a reusable pattern, API quirk, or operational rule -> write to `skills/boss-{topic}.md`
3. **Flag for attention?** -- Urgent issue, anomaly, or contradiction between what different agents reported -> tell Jaime
4. **Just observe** -- Routine conversation, no action needed

**When Jaime asks "what's going on?" or "catch me up":**
Read the activity log and MEMORY.md, then synthesize a briefing covering all channels.

## Quick API References

### Active campaigns
```bash
curl -s "https://graph.facebook.com/v21.0/act_787610255314938/campaigns?fields=id,name,status,objective,daily_budget,lifetime_budget,start_time&effective_status=%5B%22ACTIVE%22%5D&limit=100&access_token=$TOKEN"
```

### Campaign insights (last 30 days)
```bash
curl -s "https://graph.facebook.com/v21.0/{CAMPAIGN_ID}/insights?fields=spend,impressions,clicks,reach,frequency,cpm,cpc,ctr,purchase_roas,actions&date_preset=last_30d&access_token=$TOKEN"
```

### Supabase: active campaigns
```bash
curl -s "${SUPABASE_URL}/rest/v1/meta_campaigns?select=campaign_id,name,status,spend,roas,daily_budget,start_time&status=eq.ACTIVE" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```

### Supabase: TM events
```bash
curl -s "${SUPABASE_URL}/rest/v1/tm_events?order=date.asc" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```

### Discord: list channels
```bash
curl -s "$API/guilds/$GUILD_ID/channels" -H "$AUTH"
```

### Parsing Meta data
- `purchase_roas[0].value` -> ROAS (string -> float)
- Revenue = spend x ROAS
- CPA = spend / purchases (actions where action_type = "purchase")
- Budgets from Meta are in CENTS (divide by 100)

## Discord REST API (for server management)

### Create a text channel
```bash
curl -s -X POST "$API/guilds/$GUILD_ID/channels" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"name":"channel-name","type":0,"parent_id":"CATEGORY_ID","topic":"Description"}'
```

### Delete a channel
```bash
curl -s -X DELETE "$API/channels/CHANNEL_ID" -H "$AUTH"
```

### Rename/move a channel
```bash
curl -s -X PATCH "$API/channels/CHANNEL_ID" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"name":"new-name","topic":"New topic","parent_id":"NEW_CATEGORY_ID"}'
```

### Create a role
```bash
curl -s -X POST "$API/guilds/$GUILD_ID/roles" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"name":"Role Name","color":5793266}'
```

### Send an embed to a channel
```bash
curl -s -X POST "$API/channels/CHANNEL_ID/messages" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{
    "embeds": [{
      "title": "Title",
      "description": "Body text",
      "color": 5793266,
      "fields": [
        {"name": "Field 1", "value": "Value 1", "inline": true},
        {"name": "Field 2", "value": "Value 2", "inline": true}
      ],
      "footer": {"text": "Outlet Media Agent"}
    }]
  }'
```

Color reference: Brand blue = 5793266 (0x5865F2), Success = 3066993 (0x2ECC71), Error = 15158332 (0xE74C3C), Warning = 15844367 (0xF1C40F)
