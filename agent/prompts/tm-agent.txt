## Ticketmaster One (TM1)

URL: https://one.ticketmaster.com
Credentials: TM_EMAIL and TM_PASSWORD from environment or .env file.
Use the Playwright browser -- navigate and interact like a human would.

### Authentication (Cookie-based with MFA fallback)

**Step 1: Try saved cookies first.**
Load the storage state file before navigating:
```
session/tm1-storage-state.json
```
Use Playwright's `browser.newContext({ storageState: 'session/tm1-storage-state.json' })` or equivalent.
Then navigate to `https://one.ticketmaster.com`. If you land on the dashboard (page title is "TM1" and you see "Welcome, Jamie"), cookies are valid -- skip login.

**Step 2: If cookies expired (redirected to login page), do full login:**
1. Enter TM_EMAIL in the username field, click Next
2. Enter TM_PASSWORD in the password field, click Verify
3. You'll hit an email MFA step ("Get a verification email"). Click "Send me an email"
4. Click "Enter a verification code instead" to get the code input field
5. **Read the verification code from Gmail using the Gmail API:**
   ```bash
   # Get OAuth access token
   ACCESS_TOKEN=$(curl -s -X POST https://oauth2.googleapis.com/token \
     -d "client_id=$(grep GOOGLE_CLIENT_ID .env | cut -d= -f2)" \
     -d "client_secret=$(grep GOOGLE_CLIENT_SECRET .env | cut -d= -f2)" \
     -d "refresh_token=$(grep GOOGLE_REFRESH_TOKEN .env | cut -d= -f2)" \
     -d "grant_type=refresh_token" | python3 -c "import json,sys; print(json.load(sys.stdin)['access_token'])")

   # Search for the latest TM verification email (sent in last hour)
   MSG_ID=$(curl -s -H "Authorization: Bearer $ACCESS_TOKEN" \
     "https://gmail.googleapis.com/gmail/v1/users/me/messages?q=from:ticketmaster+subject:verification+newer_than:1h&maxResults=1" \
     | python3 -c "import json,sys; print(json.load(sys.stdin)['messages'][0]['id'])")

   # Extract the 6-digit code from the email body
   CODE=$(curl -s -H "Authorization: Bearer $ACCESS_TOKEN" \
     "https://gmail.googleapis.com/gmail/v1/users/me/messages/$MSG_ID?format=full" \
     | python3 -c "
   import json,sys,base64,re
   data=json.load(sys.stdin)
   body=data['payload'].get('body',{})
   text=''
   if body.get('data'): text=base64.urlsafe_b64decode(body['data']).decode('utf-8','replace')
   for p in data['payload'].get('parts',[]):
     if p.get('body',{}).get('data'):
       text=base64.urlsafe_b64decode(p['body']['data']).decode('utf-8','replace')
       break
   codes=re.findall(r'\b\d{6}\b',text)
   print(codes[-1] if codes else 'NONE')")
   ```
6. Type the code into the "Enter Code" field and click Verify
7. Wait for TM1 dashboard to load

**Step 3: After successful login, ALWAYS save the storage state:**
Use `context.storageState({ path: 'session/tm1-storage-state.json' })` to persist cookies for next time.

**Important:** The Google OAuth credentials (GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GOOGLE_REFRESH_TOKEN) must be in the .env file. The refresh token has `gmail.readonly` scope.

## TM One Sales by Location (Zip Codes)

After logging in to TM One, pull the postal code breakdown for each event using the reporting page.

**Report URL pattern:**
```
https://one.ticketmaster.com/app/reporting/type/sales-by-location/eventId/{surrogate_id}/eventStartDate/2020-01-01/eventEndDate/2029-12-31?mapViewBy=areaLevel3WithCountry
```

**Steps:**
1. Navigate to the report URL for each event (use surrogate_id from the list below)
2. Wait 4-5 seconds for the page to load (map + postal code table render asynchronously)
3. Read the "Top 10 Ticket Sales by Location" section -- find the POSTAL CODE column
4. Extract zip codes and ticket counts from the table
5. Save to session/camila-zipcodes.json (or session/last-zipcodes.json for all events)

**Output shape:**
```json
{
  "anaheim": [{"zip": "92801", "tickets": 150}, {"zip": "90001", "tickets": 120}],
  "sacramento": [{"zip": "95825", "tickets": 100}, {"zip": "95628", "tickets": 80}],
  "scraped_at": "2026-02-23T..."
}
```

**When to run:** During TM One scraping cycles. Pull zip codes for all ACTIVE/upcoming events.

**Why it matters:** Zip codes from actual buyers let us replace the blunt 25-mile radius targeting on Meta with precision zip code targeting -- cutting wasted impressions on non-converting areas.

## TM One Demographics

After logging in to TM One (session cookies active in Playwright browser), hit the demographics API for each event using the browser's fetch from the page context.

**Endpoint:**
```
GET https://one.ticketmaster.com/api/prd119/api/caui/events/demographics/info?uid=jaime%40outletmedia.net&aid=C88E27F1-109F-4BB2-8EA8-4612F5A55604&eventId={surrogate_id}
```

**Execute from the browser page** (use Playwright's `page.evaluate()` or browser_evaluate):
```js
const res = await fetch('https://one.ticketmaster.com/api/prd119/api/caui/events/demographics/info?uid=jaime%40outletmedia.net&aid=C88E27F1-109F-4BB2-8EA8-4612F5A55604&eventId={surrogate_id}', { credentials: 'include' });
const data = await res.json();
return data;
```

**Response shape:**
```json
{
  "fanInfo": { "totalFans": "1,445", "female": "55%", "married": "26%", "male": "45%", "withChildren": "59%" },
  "education": { "highSchool": "68.16%", "college": "26.84%", "gradSchool": "5%" },
  "income": { "incomeZeroToThirty": "16.29%", "incomeThirtyToSixty": "X%", "incomeSixtyToNinety": "X%", "incomeNinetyToOneTwentyFive": "X%", "incomeOverOneTwentyFive": "24.28%" },
  "payment": { "amex": "9.56%", "visa": "70.26%", "mc": "X%", "discover": "X%" },
  "age": { "ageEighteenToTwentyFour": "10.84%", "ageTwentyFiveToThirtyFour": "X%", "ageThirtyFiveToFortyFour": "X%", "ageFortyFiveToFiftyFour": "X%", "ageOverFiftyFour": "14.93%" }
}
```

**Parsing notes:**
- Strip "%" and "," before converting to float. "1,445" -> 1445 (integer). "55%" -> 55.0 (float).
- If a field is missing or null, omit it from the payload (don't send null).
- `totalFans` may be a string with commas -- parse as integer.

## Alerts Endpoint

Post alerts to the dashboard when you detect anomalies (ticket velocity stalling, status changes, upcoming show at risk):
```bash
INGEST_URL=$(grep INGEST_URL .env | cut -d= -f2)
INGEST_SECRET=$(grep INGEST_SECRET .env | cut -d= -f2)
BASE_URL="${INGEST_URL%/api/ingest}"
curl -s -X POST "$BASE_URL/api/alerts" \
  -H "Content-Type: application/json" \
  -d '{"secret":"'"$INGEST_SECRET"'","message":"[alert text]","level":"warn"}'
```
Level must be one of: "info", "warn", "error" (Zod-validated — "warning"/"critical" will be silently rejected).

## Supabase (direct query)

```bash
SUPABASE_URL=$(grep SUPABASE_URL .env | cut -d= -f2)
SUPABASE_KEY=$(grep SUPABASE_SERVICE_ROLE_KEY .env | cut -d= -f2)
```

### Events
```bash
curl -s "${SUPABASE_URL}/rest/v1/tm_events?order=date.asc" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```

### Event snapshots (daily ticket metrics)
```bash
curl -s "${SUPABASE_URL}/rest/v1/event_snapshots?tm_id=eq.{TM_ID}&order=snapshot_date.desc&limit=7" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```
Note: event_snapshots exist but TM One per-event data is currently static (identical values across dates). Velocity tracking will work once live ticket data flows.

### Campaign snapshots (for cross-referencing ad spend)
```bash
curl -s "${SUPABASE_URL}/rest/v1/campaign_snapshots?campaign_id=eq.{ID}&order=snapshot_date.desc&limit=7" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```
Column naming: `spend` (NOT `spend_cents`) — value is in cents despite the name. `roas` is a float.

## TM One Data Limitations

The Outlet Media TM One account (`app_caui` role) has restricted API access:
- Per-event info endpoint (`/events/info`): returns 200 but all ticket fields are empty/null (permission restriction)
- Demographics endpoint (`/events/demographics/info`): WORKS — returns fan data
- GraphQL events list: returns id, name, url, dates, venue, attractions — NO ticket metrics
- `tickets_sold` values currently in DB came from earlier scraper runs or manual entry
- Use the browser-based scraper approach for ticket data when needed
