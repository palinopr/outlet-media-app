## TM One Sales by Location (Zip Codes)

After logging in to TM One, pull the postal code breakdown for each event using the reporting page.

**Report URL pattern:**
```
https://one.ticketmaster.com/app/reporting/type/sales-by-location/eventId/{surrogate_id}/eventStartDate/2020-01-01/eventEndDate/2029-12-31?mapViewBy=areaLevel3WithCountry
```

**Steps:**
1. Navigate to the report URL for each event (use surrogate_id from the list below)
2. Wait 4-5 seconds for the page to load (map + postal code table render asynchronously)
3. Read the "Top 10 Ticket Sales by Location" section -- find the POSTAL CODE column
4. Extract zip codes and ticket counts from the table
5. Save to session/camila-zipcodes.json (or session/last-zipcodes.json for all events)

**Output shape:**
```json
{
  "anaheim": [{"zip": "92801", "tickets": 150}, {"zip": "90001", "tickets": 120}],
  "sacramento": [{"zip": "95825", "tickets": 100}, {"zip": "95628", "tickets": 80}],
  "scraped_at": "2026-02-23T..."
}
```

**When to run:** During TM One scraping cycles. Pull zip codes for all ACTIVE/upcoming events.

**Why it matters:** Zip codes from actual buyers let us replace the blunt 25-mile radius targeting on Meta with precision zip code targeting -- cutting wasted impressions on non-converting areas.

**All 26 surrogate IDs are listed in the Demographics section below.**

## TM One Demographics

After logging in to TM One (session cookies active in Playwright browser), hit the demographics API for each event using the browser's fetch from the page context.

**Endpoint:**
```
GET https://one.ticketmaster.com/api/prd119/api/caui/events/demographics/info?uid=jaime%40outletmedia.net&aid=C88E27F1-109F-4BB2-8EA8-4612F5A55604&eventId={surrogate_id}
```

**Execute from the browser page** (use Playwright's `page.evaluate()` or browser_evaluate):
```js
const res = await fetch('https://one.ticketmaster.com/api/prd119/api/caui/events/demographics/info?uid=jaime%40outletmedia.net&aid=C88E27F1-109F-4BB2-8EA8-4612F5A55604&eventId={surrogate_id}', { credentials: 'include' });
const data = await res.json();
return data;
```

**Response shape:**
```json
{
  "fanInfo": { "totalFans": "1,445", "female": "55%", "married": "26%", "male": "45%", "withChildren": "59%" },
  "education": { "highSchool": "68.16%", "college": "26.84%", "gradSchool": "5%" },
  "income": { "incomeZeroToThirty": "16.29%", "incomeThirtyToSixty": "X%", "incomeSixtyToNinety": "X%", "incomeNinetyToOneTwentyFive": "X%", "incomeOverOneTwentyFive": "24.28%" },
  "payment": { "amex": "9.56%", "visa": "70.26%", "mc": "X%", "discover": "X%" },
  "age": { "ageEighteenToTwentyFour": "10.84%", "ageTwentyFiveToThirtyFour": "X%", "ageThirtyFiveToFortyFour": "X%", "ageFortyFiveToFiftyFour": "X%", "ageOverFiftyFour": "14.93%" }
}
```

**Parsing notes:**
- Strip "%" and "," before converting to float. "1,445" -> 1445 (integer). "55%" -> 55.0 (float).
- If a field is missing or null, omit it from the payload (don't send null).
- `totalFans` may be a string with commas -- parse as integer.

## Ingest Endpoint

Get the URL from environment: INGEST_URL
Get the secret from environment: INGEST_SECRET

### POST TM events:
```bash
INGEST_URL=$(cat .env | grep INGEST_URL | cut -d= -f2)
INGEST_SECRET=$(cat .env | grep INGEST_SECRET | cut -d= -f2)
curl -s -X POST "$INGEST_URL" \
  -H "Content-Type: application/json" \
  -d '{"secret":"'"$INGEST_SECRET"'","source":"ticketmaster_one","data":{"events":[...],"scraped_at":"'"$(date -u +%Y-%m-%dT%H:%M:%SZ)"'"}}'
```

### POST demographics:
```bash
curl -s -X POST "$INGEST_URL" \
  -H "Content-Type: application/json" \
  -d '{
    "secret": "'"$INGEST_SECRET"'",
    "source": "tm_demographics",
    "data": {
      "demographics": [
        {
          "tm_id": "vvG1HZbw2TQALI",
          "fans_total": 1445,
          "fans_female_pct": 55.0,

          "income_0_30k_pct": 16.29,
```

## Supabase (direct query for events)

```bash
SUPABASE_URL=$(grep SUPABASE_URL .env | cut -d= -f2)
SUPABASE_KEY=$(grep SUPABASE_SERVICE_ROLE_KEY .env | cut -d= -f2)
curl -s "${SUPABASE_URL}/rest/v1/tm_events?order=date.asc" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```
