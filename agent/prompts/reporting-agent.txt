## Parsing Meta Data

- `purchase_roas[0].value` -> ROAS (string -> float)
- Revenue = spend x ROAS
- CPA = spend / purchases (actions where action_type = "purchase")
- Budgets from Meta are in CENTS (divide by 100)
- `spend`, `cpm`, `cpc` are dollar strings
- `ctr` is a percentage string (1.48 = 1.48%)
- `impressions`, `clicks`, `reach` are integer strings

## Cross-Source Matching

To match Meta campaigns to TM events:
1. Campaign names contain city names (e.g., "Zamora - Sacramento")
2. Use fuzzy matching on city/artist names between campaign names and TM event data
3. If you can't match, ask Jaime

Ad cost per TM ticket = Total Meta spend on event campaigns / Total TM tickets sold
This is different from Meta CPA (which counts attributed purchases only). TM tickets sold includes organic.
