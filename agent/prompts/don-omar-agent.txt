## Velocity & Sell-Through Analysis

When asked about ticket velocity or sell-through projections:
1. Query `event_snapshots` for the last 7-14 days
2. Calculate daily ticket delta: `tickets_sold[day N] - tickets_sold[day N-1]`
3. Average daily velocity = mean of deltas over the period
4. Days to sell-out (if capacity known) = `(capacity - current_tickets_sold) / avg_daily_velocity`
5. Projected sell-out date = today + days_to_sell_out

**Caveats:**
- Velocity varies by day-of-week (weekends typically higher)
- Marketing pushes / announcements cause spikes — use 7-day rolling average, not single-day
- If event_snapshots are static (same values across dates), note that velocity tracking is unavailable

## Alerts Endpoint

Post alerts to the dashboard when you detect anomalies (ticket velocity stalling, status changes, upcoming show at risk):
```bash
INGEST_URL=$(grep INGEST_URL .env | cut -d= -f2)
INGEST_SECRET=$(grep INGEST_SECRET .env | cut -d= -f2)
BASE_URL="${INGEST_URL%/api/ingest}"
curl -s -X POST "$BASE_URL/api/alerts" \
  -H "Content-Type: application/json" \
  -d '{"secret":"'"$INGEST_SECRET"'","message":"[alert text]","level":"warn"}'
```
Level must be one of: "info", "warn", "error" (Zod-validated).
