## EATA (Vivaticket) Platform

Platform: entradasatualcance.com (Vivaticket backstage)
Auth: OAuth2 access_token stored in session/eata-auth.json
Client slug: don_omar_bcn
Venue: Estadio Olimpico Lluis Companys, Barcelona

### Auth Details

The EATA sync uses an OAuth2 access_token that gets appended as a query parameter to all API calls.
Credentials (EATA_USERNAME, EATA_PASSWORD) are in the agent .env file.

Token refresh is handled by the scheduler (eata-cookie-refresh job) using Playwright to log in and capture a fresh token.

### API Endpoints

Base URL: https://www.entradasatualcance.com
Common query params: `access_token={token}&host=www.entradasatualcance.com&release=2.120.37&team=1578`
(All params available in session/eata-auth.json)

- `GET /api/events?...` -- list all events
- `GET /api/events/{id}?...` -- event detail
- `GET /api/events/{id}/report?...` -- sales report (revenue, tickets sold)

### Event IDs

- **14948** = DON OMAR BARCELONA (active event, the one we track)
- **14942** = NO VALIDO (skip this event in all queries/syncs)
- EATA events use `eata_{id}` as the tm_id in Supabase to prevent collisions with TM alphanumeric IDs

### Data Shape

Events are synced to the `tm_events` table with:
- `tm_id`: `eata_{vivaticket_id}` (e.g. `eata_14948`)
- `client_slug`: `don_omar_bcn`
- `artist`: `Don Omar`
- `venue`: `Estadio Olimpico Lluis Companys`
- `city`: `Barcelona`
- `tickets_sold`, `gross` (EUR), `avg_ticket_price`

### Report Data Fields (from /api/events/{id}/report)

- `revenue.total` — total revenue in whole euros
- `revenue.daily` — array of daily revenue breakdown
- `soldTickets.total` — total tickets sold
- `soldTickets.daily` — array of daily ticket sales

### Token Lifetime

OAuth2 access_token expires in ~1-2 hours. If API calls return 401/403, the token is stale.
The `eata-cookie-refresh` scheduler job refreshes it via Playwright login every 6h.
Check `session/eata-auth.json` `savedAt` timestamp to verify freshness before making API calls.

## Ingest Endpoint

Get the URL from environment: INGEST_URL
Get the secret from environment: INGEST_SECRET

### POST events:
```bash
INGEST_URL=$(grep INGEST_URL .env | cut -d= -f2)
INGEST_SECRET=$(grep INGEST_SECRET .env | cut -d= -f2)
curl -s -X POST "$INGEST_URL" \
  -H "Content-Type: application/json" \
  -d '{"secret":"'"$INGEST_SECRET"'","source":"ticketmaster_one","data":{"events":[...],"scraped_at":"'"$(date -u +%Y-%m-%dT%H:%M:%SZ)"'"}}'
```

## Supabase (direct query)

```bash
SUPABASE_URL=$(grep SUPABASE_URL .env | cut -d= -f2)
SUPABASE_KEY=$(grep SUPABASE_SERVICE_ROLE_KEY .env | cut -d= -f2)
```

### Don Omar events
```bash
curl -s "${SUPABASE_URL}/rest/v1/tm_events?client_slug=eq.don_omar_bcn&order=date.asc" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```

### Campaign snapshots (for cross-referencing ad spend)
```bash
curl -s "${SUPABASE_URL}/rest/v1/campaign_snapshots?campaign_id=eq.{ID}&order=snapshot_date.desc&limit=7" \
  -H "apikey: ${SUPABASE_KEY}" \
  -H "Authorization: Bearer ${SUPABASE_KEY}"
```

Column naming: `spend` (NOT `spend_cents`) -- value is in cents despite the name. `roas` is a float.

## Alerts Endpoint

Post alerts to the dashboard when you detect anomalies (ticket velocity stalling, status changes, upcoming show at risk):
```bash
INGEST_URL=$(grep INGEST_URL .env | cut -d= -f2)
INGEST_SECRET=$(grep INGEST_SECRET .env | cut -d= -f2)
BASE_URL="${INGEST_URL%/api/ingest}"
curl -s -X POST "$BASE_URL/api/alerts" \
  -H "Content-Type: application/json" \
  -d '{"secret":"'"$INGEST_SECRET"'","message":"[alert text]","level":"warn"}'
```
Level must be one of: "info", "warn", "error" (Zod-validated).
