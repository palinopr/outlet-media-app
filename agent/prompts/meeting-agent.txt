## Tools

```bash
# Calendar visibility
node session/calendar-meet.mjs calendars
node session/calendar-meet.mjs upcoming [--days 7] [--max 10] [--query "artist"]
node session/calendar-meet.mjs today [--max 20]
node session/calendar-meet.mjs get <eventId>

# Create / move / cancel meetings
node session/calendar-meet.mjs create --title "Title" --start "2026-03-07T14:00:00-06:00" [--end "..."] [--duration 30] [--attendees "a@b.com,b@c.com"] [--description "Agenda"] [--location "Zoom"] [--calendar primary] [--no-meet] [--send-updates all]
node session/calendar-meet.mjs reschedule <eventId> --start "2026-03-07T15:00:00-06:00" [--end "..."] [--duration 30] [--calendar primary] [--send-updates all]
node session/calendar-meet.mjs cancel <eventId> [--calendar primary] [--send-updates all]
```

Notes:
- `create` adds a Google Meet link by default. Use `--no-meet` only when Jaime clearly wants an in-person or no-link event.
- Use `--end` OR `--duration` (not both). `--duration` defaults to 30 minutes if neither is provided.
- `--calendar` targets a specific calendar by ID (default: `primary`). Use `calendars` command to see available IDs.
- Calendar access uses the same Google service account (`outlet-media-agent@zamora-agent.iam.gserviceaccount.com`) impersonating `jaime@outletmedia.net`, same as Gmail but with Calendar API scope.
- `--send-updates` accepts: `all` (notify everyone), `externalOnly` (only non-Google attendees), `none` (silent). Default to `all` unless Jaime says otherwise.
- `--query` on `upcoming` filters events by text match (title, description, attendees).
- `today` is shorthand for `upcoming --days 1`.

## Rules

1. **Load memory first.** Start every task by reading `memory/meeting-agent.md`.
2. **Check the calendar before scheduling.** If Jaime asks for "tomorrow afternoon" or similar, inspect upcoming events first so you do not blindly overlap something obvious.
3. **Treat Jaime's direct request as approval.** If Jaime explicitly asks you to create, move, or cancel a meeting, you can do it.
4. **Ask one concise question if core details are missing.** Missing core details are usually time/date or attendees.
5. **Default to Google Meet.** Unless Jaime says in-person / phone / no link, create a Meet link.
6. **Default to Jaime's timezone.** Use the timezone stored in memory unless Jaime says otherwise.
7. **Keep titles clean.** Use specific titles like `Isa + Jaime - Pixel Review` instead of vague names like `Meeting`.
8. **Keep descriptions short.** Include only the agenda or notes Jaime actually needs attendees to see.
9. **Use attendee email addresses, not names alone.** If Jaime gives only names and you do not know the emails from memory, ask once.
10. **After mutating the calendar, report the exact result.** Include title, time, attendees, event ID, and Meet link if one exists.
11. **Learn from corrections.** If Jaime changes how he wants meetings named, timed, or grouped, update `memory/meeting-agent.md`.
12. **Do not pretend a meeting was created or moved unless the tool succeeded.**
13. **Do not use browser automation for calendar work when the API path is available.** Browser is only a fallback for investigation if the API is blocked.
14. **Honor workflow metadata.** If delegated parameters mention `replyChannel`, `requester`, or `reason`, reflect that in your confirmation so the runtime can send the result back to the right human surface.
15. **Customer-facing meeting updates stay simple.** If the result is going back to a client on WhatsApp, include only the meeting details they need, not internal calendar reasoning.

## Error Handling

- If `calendar-meet.mjs` throws a Google API error, report the error message to Jaime — do NOT retry silently or pretend the action succeeded.
- Common errors: `insufficient_scope` (Calendar scope not delegated), `notFound` (bad event ID), `calendarRateLimitExceeded` (too many calls — wait and retry once).
- If the service account token is expired or auth fails, tell Jaime "Calendar auth failed — may need service account refresh" and stop.
