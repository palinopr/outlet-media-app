## Per-Ad Insights (your primary data source)

### Get all ads in a campaign with performance
```bash
curl -s "https://graph.facebook.com/v21.0/{CAMPAIGN_ID}/insights?fields=ad_id,ad_name,spend,impressions,clicks,reach,frequency,cpm,cpc,ctr,purchase_roas,actions&date_preset=last_30d&level=ad&access_token=$TOKEN"
```

### Get ad creative details
```bash
curl -s "https://graph.facebook.com/v21.0/{AD_ID}?fields=id,name,status,creative{id,title,body,image_url,thumbnail_url,video_id,object_story_spec}&access_token=$TOKEN"
```

### Get ad preview (for reviewing what the ad looks like)
```bash
curl -s "https://graph.facebook.com/v21.0/{AD_ID}/previews?ad_format=DESKTOP_FEED_STANDARD&access_token=$TOKEN"
```

### List active campaigns (to find campaign IDs)
```bash
curl -s "https://graph.facebook.com/v21.0/act_787610255314938/campaigns?fields=id,name,status&effective_status=%5B%22ACTIVE%22%5D&limit=100&access_token=$TOKEN"
```

### Parsing insights
- `purchase_roas[0].value` -> ROAS (string -> float)
- Revenue = spend x ROAS
- CPA = spend / purchases (actions where action_type = "purchase")
- CTR as percentage (1.48 = 1.48%)
- Frequency = impressions / reach. Above 3-4 = ad fatigue.

## Meta Creative Specs (2024-2026)

### Image Ads
- **Feed**: 1080x1080 (1:1) or 1080x1350 (4:5) -- 4:5 gets more real estate
- **Stories/Reels**: 1080x1920 (9:16) -- full screen vertical
- **Carousel**: 1080x1080 per card, max 10 cards
- Max file size: 30MB
- Formats: JPG, PNG
- Text overlay: keep under 20% for best delivery (not a hard rule anymore but still affects performance)

### Video Ads
- **Feed**: 1:1 or 4:5, 1-240 seconds (15-30s performs best)
- **Stories/Reels**: 9:16, max 60 seconds (under 15s for Reels)
- **In-stream**: 16:9, 5-15 seconds
- Min resolution: 1080x1080
- Max file size: 4GB
- Formats: MP4, MOV (H.264 codec preferred)
- Captions: always add -- 85% of Facebook video is watched without sound

### Ad Copy Limits
- **Primary text**: 125 chars recommended (up to 2,200 shown with "See more")
- **Headline**: 27 chars recommended (up to 255)
- **Description**: 27 chars recommended (up to 255)
- **CTA buttons**: LEARN_MORE, SHOP_NOW, BOOK_NOW, GET_TICKETS, SIGN_UP, GET_OFFER

### Concert/Event Best Practices
- Lead with the artist name and city in the headline
- Include the date prominently
- Urgency language works in final 2 weeks: "Last tickets", "This Saturday"
- Video testimonials from past shows outperform static images
- Carousel works well for multi-city tours (one card per city)
- 4:5 vertical images get more engagement than 1:1 in Feed

## Creative Analysis Framework

When analyzing creative performance:

1. **Pull per-ad data** for the campaign
2. **Rank by ROAS** (primary) and CTR (secondary)
3. **Identify top 3 and bottom 3** ads
4. **Check frequency** -- high frequency + declining CTR = fatigue
5. **Compare creative types** -- static vs video vs carousel
6. **Check spend distribution** -- is Meta over-indexing on one ad? If yes, the others may need more budget to get signal.
7. **Report**: "Top performer is [ad name] at [ROAS]x. Bottom is [ad name] at [ROAS]x. Recommendation: [action]."

### Creative Fatigue Signals
- CTR dropping week-over-week with stable or rising frequency
- CPA increasing while impressions stay high
- Frequency > 4 on any single ad

### What to recommend next
- If video outperforms static: "More video content, similar style to [top ad]"
- If carousel works: "Test carousel with different city order"
- If all creative is stale: "Full creative refresh needed -- new angles, new imagery"
- Final week before show: "Urgency creative -- countdown, last tickets messaging"

## Copy Review Checklist

When reviewing ad copy:
1. **Headline**: Artist name + city? Under 27 chars?
2. **Primary text**: Clear value prop? Urgency if close to show date?
3. **CTA**: Appropriate button (GET_TICKETS for concerts)?
4. **Spelling/grammar**: Clean?
5. **Emoji use**: Appropriate for the audience?
6. **Link preview**: Does the URL preview look good?
