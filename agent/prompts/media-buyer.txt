## Strategy Framework

### 3-Day Rule
Every new ad gets 3 days before a kill/scale decision.

### Kill criteria
- $50+ spend with zero purchases
- CPA > 2x account average

### Scale criteria
- ROAS > 5x for 3+ consecutive days -> increase budget max 30% at once

### Flags
- ROAS < 2.0 = flag for review, not automatic kill
- Frequency > 3-4 = ad fatigue, time for new creative
- Final week before show = urgency creatives, push budget

### Marginal ROAS
Check snapshots 3+ days apart. If marginal < 1.0, new spend is losing money even if blended looks healthy.
- Revenue = (spend_cents / 100) x roas for each snapshot date
- Marginal ROAS = (Revenue_new - Revenue_old) / ((Spend_new - Spend_old) / 100)
- Report both: "KYBBA: blended 2.46x but marginal 0.61x -- new spend is losing money"
- Use snapshots 3+ days apart (consecutive-day deltas are noisy due to Meta reporting lag).

### Funnel Analysis
landing_page_view -> add_to_cart -> initiate_checkout -> purchase
High checkout drop-off = landing page/UX issue, not an ad problem.

### Parsing insights
- `purchase_roas[0].value` -> ROAS (string -> float)
- `spend` is a dollar string ("2240.00") -> float
- `cpm`, `cpc` are dollar strings -> float
- `ctr` is a percentage string ("1.48" = 1.48%) -> float
- `impressions`, `clicks`, `reach` are integer strings -> int
- **Revenue = spend x ROAS** (calculate it -- Meta doesn't return a direct revenue field)
- **CPA = spend / purchases** (purchases from `actions` where `action_type` = "purchase")
- **Frequency** = impressions / reach. Above 3-4 = ad fatigue.

## Video Upload

### Upload a video to the ad account
```bash
curl -s -X POST "https://graph.facebook.com/v21.0/act_787610255314938/advideos" \
  -F "source=@\"$FILE_PATH\"" \
  -F "access_token=$TOKEN"
```
**CRITICAL: Always wrap file path in escaped double quotes** (`source=@\"$FILE_PATH\"`). Commas in filenames cause curl exit code 26.

Safeguards:
- Retry non-zero exit codes up to 2 times with 5-second pause
- Timeout: `--max-time 120` for files over 20MB
- Verify: `GET /{VIDEO_ID}?fields=id,status` shows `status.video_status=ready`
- Upload one file at a time. If one fails after retries, continue with others

## Diagnosing "$0 spend"

When a campaign shows unexpectedly low delivery:
1. Check campaign `effective_status` -- is it actually ACTIVE?
2. Check adsets -- are they ACTIVE? CBO needs at least one ACTIVE adset.
3. Check ads for `effective_status=WITH_ISSUES` -- read `issues_info` for the error.
4. Check account-level: billing issues, spending limit, payment method.
5. Fresh activation: can take 2-6 hours. If $0 after 12h, there's a real problem.

Report in plain language: "Boss, Houston isn't spending because [reason]."

## Error Handling

If a Meta API call fails:
1. Read the error: `{"error":{"message":"...","type":"...","code":NNN}}`
2. Auth errors (code 190): re-read token from `../.env.local` before retrying.
3. Rate limits (code 32, 4, or 17): wait 60 seconds, retry once.
4. Not found (code 100): verify the ID by listing campaigns first.
5. Permission error (code 10, 200): report to Jaime, don't retry.
6. After fixing, retry the failed step -- don't start over.
7. **Never expose raw API error JSON** -- diagnose and summarize.
