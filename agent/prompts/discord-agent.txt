## Channel Operations

### List all channels
```bash
curl -s "$API/guilds/$GUILD_ID/channels" -H "$AUTH"
```

### Create a text channel (type 0)
```bash
curl -s -X POST "$API/guilds/$GUILD_ID/channels" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"name":"channel-name","type":0,"parent_id":"CATEGORY_ID","topic":"Description here"}'
```
Types: 0=text, 2=voice, 4=category, 5=announcement, 13=stage, 15=forum

### Delete a channel
```bash
curl -s -X DELETE "$API/channels/CHANNEL_ID" -H "$AUTH"
```

### Rename/move/update a channel
```bash
curl -s -X PATCH "$API/channels/CHANNEL_ID" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"name":"new-name","topic":"New topic","parent_id":"NEW_CATEGORY_ID","position":0}'
```

### Set channel topic
```bash
curl -s -X PATCH "$API/channels/CHANNEL_ID" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"topic":"New topic text"}'
```

### Reorder channels within a category
```bash
curl -s -X PATCH "$API/guilds/$GUILD_ID/channels" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '[{"id":"CHANNEL_ID_1","position":0},{"id":"CHANNEL_ID_2","position":1}]'
```

## Permission Overwrites

### Set channel permissions
```bash
curl -s -X PUT "$API/channels/CHANNEL_ID/permissions/ROLE_OR_USER_ID" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{"allow":"0","deny":"2048","type":0}'
```
type: 0=role, 1=member
Common permission bits:
- VIEW_CHANNEL = 1024 (0x400)
- SEND_MESSAGES = 2048 (0x800)
- MANAGE_MESSAGES = 8192 (0x2000)
- EMBED_LINKS = 16384 (0x4000)
- ATTACH_FILES = 32768 (0x8000)
- READ_MESSAGE_HISTORY = 65536 (0x10000)
- MENTION_EVERYONE = 131072 (0x20000)
- ADD_REACTIONS = 64 (0x40)
- MANAGE_CHANNELS = 16 (0x10)

```bash

      "title": "Title",
```

## Embed Best Practices

### Dashboard panels
Post an embed, save the message ID, edit in-place when data changes. This keeps channels clean.

### Status cards
Use inline fields (max 3 per row) for compact data display:
```json
{
  "embeds": [{
    "title": "Campaign Dashboard",
    "color": 5793266,
    "fields": [
      {"name": "Active Campaigns", "value": "12", "inline": true},
      {"name": "Total Spend", "value": "$4,200", "inline": true},
      {"name": "Avg ROAS", "value": "3.2x", "inline": true}
    ]
  }]
}
```

### Rich formatting in embeds
- `**bold**` and `*italic*` work in descriptions
- `[link text](url)` for clickable links
- `> quote` for blockquotes
- ``` for code blocks
- Max 25 fields per embed, max 10 embeds per message
- Title max 256 chars, description max 4096 chars, field name 256, field value 1024

## Auto-Moderation

### Create an auto-mod rule (block words)
```bash
curl -s -X POST "$API/guilds/$GUILD_ID/auto-moderation/rules" \
  -H "$AUTH" -H "Content-Type: application/json" \
  -d '{
    "name":"Block Bad Words",
    "event_type":1,
    "trigger_type":1,
    "trigger_metadata":{"keyword_filter":["badword1","badword2"]},
    "actions":[{"type":1,"metadata":{"custom_message":"Message blocked by auto-mod"}}],
    "enabled":true
  }'
```
trigger_type: 1=keyword, 3=spam, 4=keyword preset, 5=mention spam
action type: 1=block message, 2=send alert to channel, 3=timeout member
